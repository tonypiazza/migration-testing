# OpenSearch on GKE

Deploy any supported version of OpenSearch on GKE using the [OpenSearch Kubernetes Operator](https://github.com/opensearch-project/opensearch-k8s-operator). Each deployment gets its own VPC for network isolation, making it safe for multiple users to share a single GCP project.

## Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) >= 1.5
- [gcloud CLI](https://cloud.google.com/sdk/docs/install) authenticated to your GCP project
- `kubectl`

## Quick Start

```bash
cd terraform
terraform init

terraform apply -var="project_id=YOUR_PROJECT" -var="name_prefix=os-tony"
```

After apply, kubectl is auto-configured. Retrieve credentials and test:

```bash
OS_PASSWORD=$(terraform output -raw opensearch_password)
OS_HOST=$(kubectl get svc os-target-external -o jsonpath='{.status.loadBalancer.ingress[0].ip}')

curl -k -u admin:$OS_PASSWORD "https://$OS_HOST:9200/_cluster/health?pretty"
```

## Tear Down

```bash
terraform destroy
```

## Variables

| Name | Description | Default |
|------|-------------|---------|
| `project_id` | GCP project ID | (required) |
| `name_prefix` | Prefix for resource names (e.g., your name) | `os` |
| `opensearch_version` | OpenSearch version to deploy | `2.19.0` |
| `operator_version` | OpenSearch Operator Helm chart version | `2.7.0` |
| `region` | GCP region | `us-central1` |
| `zone` | GCP zone (set to `null` for regional HA) | `us-central1-c` |
| `machine_type` | GKE node machine type | `e2-standard-4` |
| `node_count` | Number of GKE nodes | `3` |
| `disk_size_gb` | Boot disk size per node | `50` |
| `allowed_cidrs` | CIDRs allowed to reach the LoadBalancer | `["0.0.0.0/0"]` |

## Architecture

```
Terraform
  ├── VPC + Subnet (per-deployment isolation)
  ├── GKE Cluster + Node Pool
  ├── Helm: OpenSearch Operator
  └── Helm: Local chart (charts/opensearch/)
        ├── Admin credentials Secret (random password)
        ├── Security config Secret (bcrypt hash)
        ├── OpenSearchCluster CR → Operator reconciles pods
        └── LoadBalancer Service (CIDR-restricted)
```

## Security Notes

- Admin password is randomly generated by Terraform; retrieve it with `terraform output -raw opensearch_password`.
- TLS is enabled on both transport and HTTP layers (operator-generated self-signed certs).
- Use `curl -k` or extract the CA from the operator-created TLS secret.
- Restrict `allowed_cidrs` to known IPs for non-ephemeral deployments.
- `storage-rw` OAuth scope on nodes enables GCS snapshot repositories.
